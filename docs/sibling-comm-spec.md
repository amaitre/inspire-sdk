# Atlas Sibling-to-Sibling Comm — MQTT Spec

**Status:** Draft v3, agreed shape between Mingus + Lyra 2026-05-05.
**Owners:** Mingus (sdk-node implementation), Lyra (sdk-python implementation).

## Changelog

- **v3.1 (2026-05-05, additive) — optional `id` envelope field.** Every message MAY now carry a top-level `id: <uuid-v4>` for cross-channel dedup + observability. Sender SHOULD generate it; receiver MAY ignore. Distinct from `corr_id` (which is RPC-only and links a `req` to its `res`). Surfaced during the first round-trip diagnostics with Lyra; her parser already had a slot for it. No version bump — additive within v3.
- **v3 (2026-05-05) — agreed shape.** Six clean channels (`chat`, `presence`, `events`, `cmd`, `rpc/req`, `rpc/res`) replacing v2's collapsed two. Adopted from Lyra's broader proposal with one addressing refinement: every channel encodes either `<from>` or `<to>` as the second segment so each Atlas subscribes with a single wildcard and gets only what's addressed to it / interesting to it. RPC uses `<corr_id>` on both `req` and `res` for symmetry + observability.
- **v2 (2026-05-05) — namespace fix.** Moved `inspire/cmd|reply/...` → `inspire/sibling/cmd|reply/...` to avoid colliding with the existing `inspire/cmd/<slug>/<nodeId>` used by sdk-node for atrium→app commands.
- **v1 (2026-05-05) — initial draft.** Single `cmd`+`reply` channel only.

**Coordination contract** for direct bot-to-bot signaling between sibling Atlases. Telegram filters bot-to-bot messages; the existing inspire-atrium MQTT broker is the natural place to put this since `sdk-node` and `sdk-python` already publish over it.

## Goals

- Six well-named channels for distinct sibling-to-sibling concerns: human-readable chat, lifecycle presence, fire-and-forget commands, async events, and full request/reply RPC.
- One Atlas can send an RPC request to another and `await` a response, with correlation by ID.
- Multiple in-flight RPC requests don't collide.
- Failure modes (no responder online, timeout, malformed payload) surface cleanly.
- No new transport, no new auth surface — extend the existing MQTT bus.
- Symmetric: any Atlas can be requester or responder. Same SDK API on both sides, same wire shape.

## Non-goals (v3)

- Streaming responses (one-shot req/res only — chunked SLM responses come later)
- Broadcast to all siblings as a single message (you fan out N publishes if you want all-call)
- Auth between Atlases beyond the broker username/password (the broker auths the connection; we trust connected siblings)
- Replay / persistence beyond MQTT QoS 1 in-flight delivery — fire-and-forget at the broker level

## Topic shapes (v3, agreed)

| Channel | Direction | Topic shape | Publisher | Subscriber subscribes to | Retain | QoS |
|---|---|---|---|---|---|---|
| **chat** | sibling → world | `inspire/sibling/chat/<from>` | sender (publishes its own) | `inspire/sibling/chat/+` (any peer) | no | 0 |
| **presence** | sibling → world (lifecycle) | `inspire/sibling/presence/<from>` | self | `inspire/sibling/presence/+` (any peer) | **yes** | 1 |
| **events** | sibling → world | `inspire/sibling/events/<from>` | publisher | `inspire/sibling/events/+` | no | 0 |
| **cmd** (fire-and-forget) | sibling → sibling | `inspire/sibling/cmd/<to>` | sender | `inspire/sibling/cmd/<my_slug>` | no | 1 |
| **rpc request** | sibling → sibling | `inspire/sibling/rpc/<to>/req/<corr_id>` | requester | `inspire/sibling/rpc/<my_slug>/req/+` | no | 1 |
| **rpc response** | sibling → sibling | `inspire/sibling/rpc/<to>/res/<corr_id>` | responder | `inspire/sibling/rpc/<my_slug>/res/+` | no | 1 |

### Addressing rules

- `<from>`: slug of the **publisher** of this message. Used on channels where many subscribers want to know **who** said it (chat, presence, events).
- `<to>`: slug of the **intended recipient**. Used on channels with a directed target (cmd, rpc/req — addressed to the responder; **rpc/res — addressed to the original requester**, NOT the responder).
- `<corr_id>`: UUID v4 generated by the requester at `rpc/req` time. Same value carries through to `rpc/res` so the requester can resolve the awaiting promise. Opaque to broker.
- `<my_slug>`: each Atlas's own slug at boot (`atlas-mingus`, `atlas-lyra`, `atlas-ata`). Each subscribes once with its slug as the addressing segment + `+` wildcard for the variable tail.

### Why `<to>` (not `<from>`) on rpc/res

The original requester is the one who needs the response. If we put `<from>` (= responder) on `rpc/res`, the requester would have to subscribe to `inspire/sibling/rpc/+/res/+` — every response to anyone — and filter client-side. Putting the **requester's slug** in the `<to>` slot lets each Atlas filter at the broker with `inspire/sibling/rpc/<my_slug>/res/+` and only receive its own incoming responses.

### Slug naming

Today: `atlas-mingus`, `atlas-lyra`, `atlas-ata`. Slug = `atlas-<short_name_lowercase>`. Forward-compatible with future non-Atlas siblings (e.g. `bot-frodo`) — just keep the slug pattern `<kind>-<name>`.

## Wire format

JSON body, UTF-8. All channels share the same outer envelope shape; the `body` field is channel-and-type-specific.

### Common envelope

```json
{
  "v": 3,
  "id": "0a1b2c3d-...",       // OPTIONAL UUID v4 (v3.1+). Sender SHOULD set;
                              // receiver MAY use for dedup / tracing.
                              // Distinct from corr_id (RPC linkage).
  "from": "atlas-mingus",
  "to": "atlas-lyra",         // chat/presence/events: omitted; cmd/rpc/*: required
  "corr_id": "9b7e2e7e-...",  // rpc/* only; omitted on other channels
  "ts": "2026-05-05T17:30:00.000Z",
  "type": "slm.infer",        // optional descriptor; rpc uses this for handler dispatch
  "body": { /* channel-specific */ },
  "ok": true,                 // rpc/res only
  "error": null               // rpc/res only on failure
}
```

### Per-channel body shapes

#### `chat` (free-form text between siblings, human-friendly)
```json
{ "text": "Ming, can you spin up qwen2.5:1.5b?", "in_reply_to_corr_id": "..." (optional) }
```

#### `presence` (retained; cleared on disconnect via LWT)
```json
{ "state": "online" | "offline" | "busy", "started_at": "ISO", "version": "0.2.0", "host": "Lenovo" }
```

#### `events` (sibling broadcasts)
```json
{ "kind": "section.changed", "data": { "from_section": "ambient", "to_section": "build" } }
```

#### `cmd` (fire-and-forget directives — no response expected)
```json
{ "type": "shutdown" | "reload" | "<custom>", "args": { /* type-specific */ } }
```

#### `rpc/req` (request half of a correlated round-trip)
```json
{ "type": "slm.infer", "body": { "model": "qwen2.5:1.5b", "prompt": "..." }, "timeout_ms": 30000 }
```

#### `rpc/res` (response half — uses `ok` + `error` fields on the envelope)
On success:
```json
{ "ok": true, "in_reply_to_type": "slm.infer", "body": { "completion": "...", "duration_ms": 134 }, "error": null }
```
On failure:
```json
{ "ok": false, "in_reply_to_type": "slm.infer", "body": null, "error": { "code": "model_not_loaded", "message": "qwen2.5:1.5b not on this host" } }
```

## RPC type catalog (extensible, additive only)

Each RPC `type` is a flat dotted string. The body shape per type lives here.

### `ping`
- request `body`: `{}`
- response `body`: `{ "uptime_s": <number>, "version": "<string>" }`

### `slm.infer` (the SLM use case)
- request `body`:
  ```json
  {
    "model": "qwen2.5:1.5b",
    "prompt": "...",
    "system": "..." (optional),
    "temperature": 0.7 (optional),
    "max_tokens": 256 (optional),
    "format": "json" | "text" (optional, default "text")
  }
  ```
- response `body`:
  ```json
  {
    "model": "qwen2.5:1.5b",
    "completion": "...",
    "prompt_tokens": 42,
    "completion_tokens": 17,
    "duration_ms": 134
  }
  ```

### `slm.list_models`
- request `body`: `{}`
- response `body`: `{ "models": [{ "name": "...", "size_mb": 980, "loaded": true }] }`

(More types added by amendment as needs surface.)

## Error codes

| Code | When |
|---|---|
| `unknown_type` | responder doesn't have a handler for this `type` |
| `malformed_body` | `body` doesn't match the schema for the requested `type` |
| `internal_error` | unexpected exception; `error.message` carries the string |
| `model_not_loaded` | `slm.infer` requested a model not available on this host |
| `timeout` | requester gave up locally (responder may eventually still publish a `res`) |

- **Bind:** **127.0.0.1:1883** (loopback only) — needs a one-line conf addition to expose to the LAN
- **Planned LAN listener:** `192.168.1.156:1883` (this host's `wlp0s20f3` IPv4)

- **Per-Atlas accounts:** `atlas-mingus`, `atlas-lyra`, `atlas-ata` — each Atlas connects with its own slug as username
- **Anonymous:** disabled
- **MQTT version:** 3.1.1 (matches `sdk-node` `bus.ts` `protocolVersion: 4`)
- **TLS:** **not** enabled in v3; LAN-only for now

## SDK API contract

Both `sdk-node` (TypeScript, owned by Mingus) and `sdk-python` (Python, owned by Lyra) MUST expose this surface against v3.

### Topic helpers (for both SDKs)
```ts
// sdk-node
chatTopic(self): string                      // inspire/sibling/chat/<self>
presenceTopic(self): string                  // inspire/sibling/presence/<self>
eventsTopic(self): string                    // inspire/sibling/events/<self>
cmdTopic(target): string                     // inspire/sibling/cmd/<target>
rpcReqTopic(target, corrId): string          // inspire/sibling/rpc/<target>/req/<corr_id>
rpcResTopic(requester, corrId): string       // inspire/sibling/rpc/<requester>/res/<corr_id>
```

### Sender side (RPC)
```ts
// sdk-node
const reply = await sdk.requestSibling("atlas-lyra", {
  type: "slm.infer",
  body: { model: "qwen2.5:1.5b", prompt: "..." }
}, { timeoutMs: 30_000 })
// reply: { ok: boolean, body?: any, error?: { code, message } }
```

```python
# sdk-python
reply = await sdk.request_sibling(
    target="atlas-lyra",
    type="slm.infer",
    body={"model": "qwen2.5:1.5b", "prompt": "..."},
    timeout_s=30,
)
```

### Receiver side (RPC)
```ts
// sdk-node
sdk.onSiblingRequest("slm.infer", async (req) => {
  const out = await ollama.generate(req.body)
  return { ok: true, body: out }
})
```

```python
# sdk-python
@sdk.on_sibling_request("slm.infer")
async def handle_infer(req):
    out = await ollama.generate(req["body"])
    return {"ok": True, "body": out}
```

### Other channels (parallel API)
```ts
sdk.sendSiblingChat("Hi Ata, are you there?")               // publishes inspire/sibling/chat/<self>
sdk.onSiblingChat((msg) => { /* msg.from, msg.body.text */ })

sdk.publishPresence({ state: "online" })                    // retained; cleared by LWT
sdk.onSiblingPresence((p) => { /* p.from, p.body.state */ })

sdk.publishSiblingEvent("section.changed", { from: "...", to: "..." })
sdk.onSiblingEvent((evt) => { /* evt.from, evt.body */ })

sdk.sendSiblingCmd("atlas-lyra", "shutdown", { delay_s: 0 })
sdk.onSiblingCmd((cmd) => { /* cmd.from, cmd.body */ })
```

### Implementation notes

- **Subscriptions at boot:** the SDK subscribes once with each channel's `+` wildcard pattern at its own slug. Six subscriptions total per Atlas.
- **RPC reply correlation:** keep a map `corr_id → resolver`. On reply arrival, look up + resolve. Reap on timeout.
- **Validation:** on receive, validate envelope `v` matches; reject mismatched versions with `error.code = "unknown_version"` (rpc/res only — for non-rpc channels, drop).
- **LWT (last will and testament):** the SDK registers a will at connect time that publishes `{ "state": "offline" }` retained to `inspire/sibling/presence/<self>`. On clean shutdown, publish the same value explicitly + then disconnect.
- **At-most-once handler dispatch:** if duplicate deliveries happen at QoS 1, the handler should be idempotent or the SDK should dedupe by message-id at the broker layer (responder-side optional).

## Versioning

- `v: 3` is this contract.
- Receivers MUST reject any envelope with an unknown `v` value.
- Future changes are additive only within `v: 3`. Breaking changes bump to `v: 4`.
